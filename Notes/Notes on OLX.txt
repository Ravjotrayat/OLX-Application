About Monolithic Architecture
=> M app. has single codebase(Single repository) with multiple modules. Modules can be either business or technical feature.
	It has a single build system which build entire application / dependency.

About Service Oriented Architecture(SOA)
=> In-shot "module based service separation"

About Microservice Architecture
=> It is a collections of services that are: loosely coupled, highly maintainable and testable, independently deployed, owned by small teams

=> Webservices allows us to establish language and platform independent communication.
Types of Webservices::
- SOAP(Completely based upon XML)
- REST(Representational State Transfer)
	(protocol = http, payload support = XML, JSON, HTML, TEXT)
REST API => URL + HTTP Method(GET, POST, PUT, DELETE)

Actuator :
=> You can modify the base path from (http://localhost:8080/actuator) to (http://localhost:8080/Advertise)
=> We can enable all/particular endpoints of actuator.
=> We can override the existing as well as create new endpoints actuator using Builder class, on condition :
	- for @GetMapping we have --> @ReadOperation
	- for @PostMapping we have --> @WriteOperation
	- for @DeleteMapping we have --> @DeleteOperation
	- for @PathVariable we have --> @Selector
